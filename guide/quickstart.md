# Quickstart

This quickstart is for someone discovering ClaudeLimit Planner from GitHub, a search result, a comparison article, or a product directory.

## 1. Open The Hosted SaaS

Start here:

- https://claudelimitplanner.space/?utm_source=github&utm_medium=documentation&utm_campaign=claudelimit_planner_docs&utm_content=quickstart_home

The first visit should answer three questions:

- Does the first screen make the Claude Code and Codex quota planner workflow obvious?
- Is there a usable preview or concrete product surface before pricing?
- Is the Team annual path easy to find?

## 2. Learn The Workflow

1. Estimate the work, deadline, repo risk, and reviewer availability.
2. Map Claude, Codex, Gemini, and human-review lanes to available quota windows.
3. Split large AI coding work into recoverable batches with test expectations.
4. Export a weekly capacity brief before the team burns premium model time.

## 3. Compare Against Your Current Process

Use this simple evaluation prompt internally:

```text
Where does ClaudeLimit Planner reduce repeated manual work?
What private data would it touch?
Who approves the first rollout?
What evidence do we need before paying for the default plan?
```

## 4. Review Pricing

Open the tracked pricing link:

- https://claudelimitplanner.space/pricing/?utm_source=github&utm_medium=documentation&utm_campaign=claudelimit_planner_docs&utm_content=quickstart_pricing

The default buyer path is Team annual. If a different plan is more appropriate, document why before checkout.
